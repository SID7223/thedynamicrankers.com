POST-DEPLOYMENT DATABASE INITIALIZATION & POPULATION
====================================================

Follow these exact steps to move from the old "immature" database to the new
"Industry-Level" standard.

STEP 1: FLUSH PREVIOUS DATA
---------------------------
Run this command to wipe the old, inconsistent schema.
This clears the way for the professional UUID foundation.

npx wrangler d1 execute dr-command-center --remote --command="
PRAGMA foreign_keys = OFF;
DROP TABLE IF EXISTS message_search_index;
DROP TABLE IF EXISTS message_mentions;
DROP TABLE IF EXISTS message_attachments;
DROP TABLE IF EXISTS message_reads;
DROP TABLE IF EXISTS messages;
DROP TABLE IF EXISTS task_permissions;
DROP TABLE IF EXISTS chat_rooms;
DROP TABLE IF EXISTS task_access_requests;
DROP TABLE IF EXISTS tasks;
DROP TABLE IF EXISTS users;
DROP TABLE IF EXISTS crm_invoices;
DROP TABLE IF EXISTS crm_appointments;
DROP TABLE IF EXISTS crm_customers;
DROP TABLE IF EXISTS notifications;
PRAGMA foreign_keys = ON;"


STEP 2: POPULATE NEW INDUSTRY SCHEMA
------------------------------------
Run this command. It uses the 'consolidated_schema.sql' file to create all
tables and insert the initial superusers and the global command room.

npx wrangler d1 execute dr-command-center --remote --file=consolidated_schema.sql


STEP 3: VERIFY SEED DATA (MANUAL CHECK)
---------------------------------------
Run these two commands to ensure the system is ready for login:

1. Check Users:
npx wrangler d1 execute dr-command-center --remote --command="SELECT name, email FROM users;"

2. Check Global Room:
npx wrangler d1 execute dr-command-center --remote --command="SELECT id, type FROM chat_rooms WHERE type='global';"
